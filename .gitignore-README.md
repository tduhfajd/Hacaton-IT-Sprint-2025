# .gitignore Configuration Guide

This project uses a comprehensive `.gitignore` setup to ensure only necessary files are tracked by Git while keeping sensitive information secure.

## 📁 File Structure

```
it-support/
├── .gitignore                    # Main .gitignore (this file)
├── .env.example                  # Environment variables template
├── gigachat-config.example.json  # GigaChat configuration template
├── backend/.gitignore            # Backend-specific ignores
├── frontend/.gitignore           # Frontend-specific ignores
├── database/.gitignore           # Database-specific ignores
├── .specify/.gitignore           # Specify workflow ignores
├── knowledge_base/.gitignore     # Knowledge base ignores
└── .gitignore-README.md          # This documentation
```

## 🔒 Security & Secrets

### What's Ignored
- **Environment files**: `.env`, `.env.*` (except `.env.example`)
- **API credentials**: `gigachat-config.json`, `secrets/`
- **SSL certificates**: `*.crt`, `*.key`, `*.pem` (except examples)
- **Database credentials**: `database-config.json`, `db-credentials.json`

### What's Kept
- **Templates**: `.env.example`, `gigachat-config.example.json`
- **Examples**: `example-*.crt`, `example-*.key`

## 🚀 Technology-Specific Ignores

### Node.js & NPM
- `node_modules/` - Dependencies
- `npm-debug.log*` - Debug logs
- `*.tsbuildinfo` - TypeScript cache

- `*.pyc` - Compiled Python files
- `.venv/` - Virtual environments

### Docker
- `docker-data/` - Docker volumes
- `docker-compose.override.yml` - Local overrides

### Database
- `*.sqlite`, `*.db` - Database files
- `*.dump` - Database dumps

## ⚙️ Configuration

### Environment Setup
1. Copy `.env.example` to `.env`
2. Fill in your actual values
3. Copy `gigachat-config.example.json` to `gigachat-config.json`
4. Update with your GigaChat credentials

### GigaChat Setup
```bash
# Install Ministry certificates
sudo mkdir -p /usr/local/share/ca-certificates/mindigital
sudo cp certs/ministry/*.crt /usr/local/share/ca-certificates/mindigital/
sudo update-ca-certificates
```

## 🔍 Checking Ignored Files

### List ignored files
```bash
git status --ignored
```

### Check if file is ignored
```bash
git check-ignore -v path/to/file
```

### Force add ignored file (use with caution)
```bash
git add -f path/to/file
```

## 📝 Adding New Ignores

### Project-wide
Add patterns to the main `.gitignore` file.

### Directory-specific
Add patterns to the appropriate `.gitignore` file in that directory.

### Temporary
Use `git update-index --skip-worktree <file>` for temporary ignores.

## ⚠️ Important Notes

1. **Never commit secrets** - Use `.env.example` templates
2. **Keep examples** - Include `example-*` files for reference
3. **Test changes** - Verify ignores work as expected
4. **Document patterns** - Add comments for complex patterns
5. **Review regularly** - Update ignores as project evolves

## 🆘 Troubleshooting

### File still tracked after adding to .gitignore
```bash
# Remove from Git but keep locally
git rm --cached path/to/file
git commit -m "Remove tracked file, now ignored"
```

### Need to ignore already tracked files
```bash
# Stop tracking but keep locally
git rm -r --cached .
git add .
git commit -m "Apply .gitignore to tracked files"
```

### Check what would be ignored
```bash
# Dry run to see what would be ignored
git clean -n
```
