# 📝 Changelog

Все заметные изменения в проекте SmartSupport будут документированы в этом файле.

## [2.0.0] - 2025-10-25

### 🎉 Добавлено

#### Telegram интеграция
- **Telegram бот** для приёма обращений от граждан
- **Выбор категории** через inline-кнопки в Telegram
- **Двусторонняя связь**: сообщения от граждан → панель оператора, ответы оператора → Telegram
- **Хранение сессий** в Redis (категория, ID обращения, состояние)
- **Автоматические уведомления** в Telegram при завершении обращения

#### Умное управление статусами
- **Автоматический переход** `Новое` → `В работе` при открытии оператором
- **Автоматический переход** `В работе` → `Новое` при новом сообщении от гражданина
- **Счётчик непрочитанных** сообщений с визуальным бейджем 🔴
- **Отслеживание активности**: поле `last_activity_at` в таблице appeals
- **Автозакрытие чата** после отправки сообщения оператором

#### Автоматизация
- **Celery Beat** для планирования периодических задач
- **Автозавершение неактивных обращений** (задача каждые 2 минуты)
- **Завершение через 5 минут** неактивности с уведомлением в Telegram
- **Миграция БД** для добавления полей `unread_operator_count` и `last_activity_at`

#### ИИ-улучшения
- **Фильтрация "пустых" сообщений**: приветствия, благодарности, прощания игнорируются ИИ
- **Контекстные ответы**: ИИ учитывает историю диалога
- **Улучшенная генерация**: более человекоподобные ответы

#### Инфраструктура
- **Единый docker-compose.yml** для всех сервисов
- **Единая сеть nginx-net** для всех контейнеров
- **Централизованный .env.prod** файл
- **Правильные зависимости** между сервисами

### 🔧 Изменено

#### Backend
- Отключена авторизация для панели оператора и базы знаний (для тестирования)
- `ChatService.ts`: добавлена логика автозакрытия чата и управления статусами
- `TelegramBotService.ts`: создан новый сервис для интеграции с Telegram
- Упрощена валидация UUID для поддержки демо-оператора

#### Frontend (Operator)
- Удалён компонент `LoginForm.tsx` (авторизация отключена)
- `ChatWindow.tsx`: добавлен обработчик события `close_chat` для автозакрытия
- `App.tsx`: исправлена работа с WebSocket событиями `appeal_updated`
- Отключён PWA (Service Worker) для предотвращения проблем с кешированием
- Изменён механизм обновления ИИ-рекомендаций (polling с 3 попытками)

#### Database
- Добавлена миграция `006_add_activity_tracking.sql`:
  - `unread_operator_count INTEGER DEFAULT 0`
  - `last_activity_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()`
  - Индекс на `last_activity_at`
- Добавлена миграция `003_add_telegram_support.sql`:
  - `telegram_chat_id BIGINT`
- Добавлена миграция `004_add_in_progress_status.sql`:
  - Новый статус `in_progress` в enum `appeal_status`

#### Workers
- `celery_app.py`: добавлен `beat_schedule` для автозавершения обращений
- `auto_complete_appeals.py`: новая задача для автозавершения
- `contextual_response.py`: добавлена фильтрация "пустых" сообщений

### 🐛 Исправлено

- **Автозакрытие чата**: теперь работает стабильно при повторных ответах
- **WebSocket обработчики**: корректная очистка при размонтировании компонента
- **Кеширование браузера**: отключён PWA для предотвращения проблем с обновлениями
- **Очереди Celery**: исправлена конфигурация для корректной обработки задач
- **UUID валидация**: поддержка демо-оператора с ID `00000000-0000-0000-0000-000000000001`
- **Обновление статусов**: корректная синхронизация между фронтендом и бэкендом

### 📚 Документация

- **README.md**: добавлена секция "Логика работы системы"
- **README.md**: добавлено описание Telegram интеграции
- **README.md**: обновлена архитектурная схема
- **README.md**: добавлена таблица управления статусами
- **README.md**: обновлён раздел "Использование" с инструкциями для Telegram
- **QUICKSTART.md**: создан новый файл с кратким руководством
- **CHANGELOG.md**: создан этот файл для отслеживания изменений

### ⚠️ Важные изменения

- Авторизация отключена для панели оператора и базы знаний (только для тестирования!)
- Обращения теперь автоматически завершаются через 5 минут неактивности
- Чат оператора автоматически закрывается после отправки сообщения
- Все сервисы теперь запускаются одним `docker-compose.yml` файлом

## [1.0.0] - 2025-10-10

### 🎉 Первый релиз

- Веб-интерфейс для граждан
- Панель оператора
- Панель администратора (база знаний)
- Backend API с WebSocket
- GigaChat интеграция для ИИ-анализа
- PostgreSQL база данных
- Redis кеширование
- RabbitMQ + Celery для асинхронных задач
- Docker Compose для развёртывания
- Nginx-proxy для автоматического HTTPS
